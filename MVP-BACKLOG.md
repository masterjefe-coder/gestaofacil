# MVP Backlog

## P0

- definir identidade e landing positioning
- criar autenticacao
- criar modelo de workspace
- criar cadastro da empresa
- criar cadastro de clientes
- criar cadastro de servicos
- criar pipeline comercial simples
- criar geracao de orcamento
- criar pagina de aprovacao de orcamento por link
- converter orcamento em pedido
- criar status de pedido
- integrar cobranca Pix ou link
- criar painel de cobrancas
- criar dashboard inicial
- estruturar emissao de NFS-e
- criar trilha de auditoria basica

## P1

- lembretes automaticos de cobranca
- historico comercial por cliente
- IA para resumo de atendimento
- IA para descricao de servico
- tags e filtros no pipeline
- relatorio simples de vendas e recebimentos
- onboarding guiado

## P2

- integracao nativa com WhatsApp
- automacoes de reativacao
- recorrencia de cobranca
- multiplos usuarios com permissoes
- area do contador ou parceiro
- score de clientes parados

## Fora do MVP

- estoque avancado
- contas a pagar completo
- ERP financeiro amplo
- NF-e como modulo principal
- multiempresa complexa
- BI avancado
- omnichannel completo

## Primeira versao que precisa vender

A primeira versao precisa permitir:

1. cadastrar cliente
2. criar orcamento
3. aprovar venda
4. cobrar
5. acompanhar pagamento
6. emitir NFS-e

Se isso funcionar muito bem, o produto ja tem valor.
